# Accessibility Audit

Use when a web UI or component needs to meet WCAG 2.2 Level AA, or before shipping a
user-facing flow.

## Capabilities
- Keyboard operability: focus order, visible focus, no keyboard traps, skip links.
- Screen-reader semantics: correct roles/names, landmark structure, live regions.
- ARIA: when NOT to use ARIA, fixing redundant/contradictory attributes.
- Color contrast (text and non-text), target size, and reduced-motion support.
- Forms: label association, error identification, and programmatic state.

## Notes
Standards-based audit with concrete remediations, distinct from general UX critique.
